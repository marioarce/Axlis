# Security Policy

## Supported Versions

| Version | Supported |
|---|---|
| 0.1.x (prerelease) | ✅ Active development |

Once a stable `1.0.0` is released, this table will be updated to reflect the support window.

## Reporting a Vulnerability

**Please do not report security vulnerabilities through public GitHub Issues.**

To report a vulnerability, use [GitHub's private vulnerability reporting](https://github.com/marioarce/Axlis/security/advisories/new) or email the maintainer directly (see the GitHub profile for contact information).

Please include as much of the following information as possible to help understand and reproduce the issue:

- Type of issue (e.g. injection, path traversal, credential exposure)
- Affected package(s) and version(s)
- Step-by-step instructions to reproduce
- Proof-of-concept code or exploit (if available)
- Impact assessment

You should receive a response within **72 hours**. If you do not, please follow up.

## Disclosure Policy

We follow [responsible disclosure](https://en.wikipedia.org/wiki/Coordinated_vulnerability_disclosure): we will work with you to understand and remediate the issue before public disclosure. We aim to release a fix within **14 days** of confirmation.
